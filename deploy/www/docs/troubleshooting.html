Troubleshooting

Enrollment: token rejected

  • Token expired — issue a new one with shorter TTL if handoff is slow
  • Hostname mismatch — token hostname must equal disk enroll --hostname
  • Token already used — tokens are single-use

Daemon won't start before enroll

Expected: client.crt is created during enrollment. Complete enroll first, then systemctl restart disk-arcana (Linux).

Files not syncing

  • Check selective sync includes — path may be excluded
  • Verify share is initialized: disk share list
  • Inspect local status: curl -sf http://127.0.0.1:9444/status
  • Dashboard Last seen should update within ~1 minute

Dashboard API errors / 401

  • Session expired — log in again (refresh token runs automatically on 401)
  • Dev override: open dashboard with ?api=https://your-host:9446
  • Reverse proxy must forward /auth/* and /dashboard/* to the health listener

Permission denied on download (selective sync)

Path is outside configured include prefixes for that device. Update includes in dashboard Selective sync or clear includes to sync full vault.

Windows-specific

See docs/windows-platform-notes.md in the GitHub repo for path length, service account, and firewall notes.
