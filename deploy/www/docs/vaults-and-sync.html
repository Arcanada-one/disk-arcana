Vaults and sync model

Vaults

A vault is a named sync namespace (vault ID, e.g. default or wiki). Vaults belong to your tenant and are created when you run disk share init on an enrolled device.

The server routes sync traffic by vault ID (HTTP header x-disk-share / gRPC metadata).

Shares

A share maps a local directory on a device to a vault ID. Multiple devices can sync the same vault; the engine reconciles file metadata with vector clocks and handles renames via stable inode identity.

Selective sync

Configure per-device folder include prefixes in the dashboard or via API/CLI:

  • Empty includes → sync the full vault
  • Non-empty → only paths matching a prefix (or under prefix/) sync
  • Enforced on exchange_state, DeltaDownload, and DeltaUpload

Sharing & roles

RoleCapabilities
OwnerFull read/write; manage invites and trash purge
EditorRead/write versions, snapshots, trash restore
ViewerRead-only lists and downloads

Trash / recycle bin

Deleted files move to trash with tier-based retention (Free / Pro / Team). Restore from dashboard or disk trash restore. Permanent delete and empty-bin require owner role on shared vaults.

CLI quick reference

disk share init --preset obsidian-vault
disk selective-sync set --vault default --node my-laptop --include docs,photos
disk sharing invites create --vault wiki --role viewer
disk trash list --vault default
