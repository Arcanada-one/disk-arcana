Enroll a device

Enrollment binds your machine to your tenant with an mTLS client certificate. Tokens are single-use, hostname-bound, and expire quickly.

1. Issue a token (operator)

From a host that already has admin access:

disk admin pending-token \
  --server https://disk.arcanada.ai:9445 \
  --hostname my-laptop \
  --ttl-secs 600

Copy the token= value. Default enrollment listener is :9445 (TLS, no client cert). Production sync uses :9443 (mTLS) after enrollment.

2. Enroll on the target machine

disk enroll \
  --server https://disk.arcanada.ai:9445 \
  --token <hex-token> \
  --hostname my-laptop

On success the client writes client.crt / client.key and starts the daemon.

3. Verify in the dashboard

Open dashboard → Devices. Your node ID and platform should appear with a recent Last seen time after the daemon connects.

Common mistakes

  • Hostname in the token must match --hostname on enroll
  • Expired or already-used tokens are rejected — issue a fresh token
  • Clock skew > 5 minutes can break TLS — sync system time
